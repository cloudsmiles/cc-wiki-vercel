## Shader脚本
```
Shader "Youtube/Stencil" {

    Properties {

        [IntRange] _StencilID ("Stencil ID", Range(0, 255)) = 0

    }

    SubShader {

        Tags {

            "RenderType"="Opaque"

            "RenderPipeline"="UniversalPipeline"

            "Queue"="Geometry"

        }

        Pass {

            Blend Zero One

            ZWrite Off

  

            Stencil {

                Ref [_StencilID]

                Comp Always

                Pass Replace

                Fail Keep

            }

        }

    }

}
```
- 队列设置为Geometry，是因为render feature可以重新在afterRendering的流程里，重新渲染，所以不需要更改队列
- 此shader的作用，是在渲染的物体上蒙上一层模板，改写stencil buffer的值，需要结合render feature使用

## Render Feature
利用unity的render feature

1. 添加Layer
![[Pasted image 20241025001122.png]]

2. 在设置里，过滤指定层，此时界面会消失该层的物体
![[Pasted image 20241025000423.png]]

3. 新建Render Feature，选择Render Objects，选择Event为AfterRenderingOpaques，Queue为Opaque，LayerMask为StencilLayer，在Overrides里勾选stencil，选择value=1（与Material的property设置的值一样），Compare Function为1
![[Pasted image 20241025001230.png]]
4. 此时就能重新渲染stencil buffer值为1的元素，也就是StencilLayer层的物体，这样做的目的是，如果有多个层的物体，可以相互各自显示，而且只能通过模板的物体去展示


## 总结
改写物体在stencil buffer的值，然后通过模板测试，就可以指定显示自己想要显示的，屏蔽不想显示的，可以实现一种神奇的视觉效果，比如平面里面存在立体的东西
