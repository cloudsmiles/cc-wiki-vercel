# 导航过程

[Inside look at modern web browser (part 2) - Chrome Developers](https://developer.chrome.com/blog/inside-browser-part2/)

- Handling input
    
    UI线程判断是URL还是搜索关键字
    
- Start navigation
    
    网络线程，DNS查找，建立TLS连接
    
- Read response
    
    网络线程，Content-Type表明返回的数据类型
    
    如果是HTML文件，会把数据转交到渲染进程
    
- Find a renderer process
    
    data is ready
    
- Commit navigation
    
    Now that the data and the renderer process is ready, an IPC is sent from the browser process to the renderer process to commit the navigation.
    
    Once the browser process hears confirmation that the commit has happened in the renderer process, the navigation is complete and the document loading phase begins
    
- Extra: **Initial load complete**
    
    当 renderer process 渲染结束（渲染结束意味着该页面内的所有的页面，包括所有 iframe 都触发了 onload 时），会发送 IPC 信号到 Browser process， UI thread 会停止展示 tab 中的 spinner
