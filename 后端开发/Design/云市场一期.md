# 云市场一期

## 需求背景

- 目前基于ZEGO业务，内部已接入和包装了部分第三方插件/能力，提供一站式解决方案，比如数美审核、讯飞/谷歌转译、大饼音频特效等，但目前**缺少统一的对外宣介载体，新客户可能不知悉即构已有这些合作伙伴和能力**。因此，做到官网云市场，客户可以一目了然，也可作为公司品牌背书；
- **借此机会从整体视角重新审视和梳理之前的开通配置、计费、用量、账单等流程，并将第三方插件开放给客户完全标准化和自助化，从而规范内部流程提高运营效率，优化客户体验。**

2. **云产品元信息表 cloud_market_product**

| 名称 | 类型 | 说明 |
| --- | --- | --- |
| id | int | 自增主键 |
| name | string | 名称 |
| product_type | string | shumei-数美审核 |
| created_at | int | 创建时间 |
| updated_at | int | 更新时间 |
1. 用于记录云产品的基本信息
2. product_type为云产品的标识
3. **云产品账号和服务关系表 cloud_market_account**

| 名称 | 类型 | 说明 |
| --- | --- | --- |
| id | int | 自增主键 |
| s_id | bigint | 客户id |
| product_id | int | 云产品id |
| config_id | int | 配置id |
| state | tinyint | 服务状态1-待开通10-已开通90-已关闭 |
| trail_endtime | bigint | 试用结束时间 |
| deadline | bigint | 截止时间套餐包耗尽后，3天缓冲期的结束时间 |
| created_at | bigint | 创建时间 |
| updated_at | bigint | 更新时间 |
1. 每个云产品开通会创建一个唯一的账号记录（sid+product_id）
2. 数美是根据账号维度开启和关闭服务，所以需要记录配置中心的配置id（config_id），通过它来调用配置中心接口
3. 账号下同一类别服务，其中一个app开通，账号的服务状态为开通；所有app关闭，账号的服务状态为关闭
4. **云产品和app关系表 cloud_market_app**

| 名称 | 类型 | 说明 |
| --- | --- | --- |
| id | int | 自增主键 |
| product_id | int | 云产品id |
| app_id | int | 应用id |
| state | tinyint | 应用状态1-待开通10-已开通90-已关闭 |
| created_at | bigint | 创建时间 |
| updated_at | bigint | 更新时间 |
1. app绑定云产品，会创建一个云产品app记录（app_id+product_id）
