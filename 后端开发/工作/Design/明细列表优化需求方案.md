[[需求] 【收支明细】充值明细中增加对公转账、账单扣费、套餐包购买的显示和收据](https://project.feishu.cn/ibc/story/detail/4285807)

1. 收支明细接口

现有接口 /v2/online_sale/get_topup_record 只返回了充值明细的数据，并且是从控制台本地库查的。

新需求需要返回充值明细、公转账、账单扣费、套餐包购买等场景的数据，需统一从晖哥的接口查。

**接口名： /v2/proxy/do_action（透传参数）**

请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| page |  | 当前页数 |
| page_size |  | 每页条数 |
| date_begin |  | 开始时间 |
| date_end |  | 结束时间 |

返回参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| Time |  | 当前收支明细发生的时间，按照UTC+ 8的时间显示（即财务入账时间） |
| 订单号 |  | 为BOSS系统中的订单ID，没有则显示"-" |
| 交易类型 |  | 分为在线充值 (PayPal)，在线充值 (Stripe)，Bank Transfer，日结账单预扣费，套餐包充值，套餐包消费，月度扣费（暂时不展示） |
| 交易金额 |  | 有正负数，保留小数点后两位 |
1. 收质明细接口(收据)

1、目前牵扯到用户收据行为的有套餐包购买、在线充值和对公转账，目前均有对应的邮件通知，需要在邮件通知模板中增加对应的Receipt --- 是指控制台发的邮件。

> 如果支付方式为对公转账，前端直接取列表上的内容
> 

**接口名： /v2/bill/get_payment_receipt**

请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| 订单号 |  | 为BOSS系统中的订单ID |

返回参数

| 字段 |  | 类型 | 描述 |
| --- | --- | --- | --- |
| 订单号 |  | string | 为BOSS系统中的订单ID |
| Date & Time |  | string | 当前收支明细发生的时间，按照UTC+ 8的时间显示（即财务入账时间） |
| PaymentList |  | array |  |
|  | 序号 |  |  |
|  | 交易类型 |  | 交易类型按照不同类型显示-在购买套餐包时，显示控制台订单的商品名称，具体商品名称见右表-在线充值时显示为Top up-对用转账时显示为Transfer |
|  | 支付方式 |  | Stripe, PayPal, Banktransfer |
|  | 金额 |  | 保留小数点后两位 |
1. 个人信息页面新增发票抬头字段

按晖哥的接口，直接透传给manage，控制台不需要存。

确认使用修改公司接口中的invoice_title字段

修改公司接口：http://api-mgr.zego.cloud/project/219/interface/api/5657
