# 销售系统v1.5方案

## 需求

> 需求说明V1.5 跟进及销售项目管理优化
> 

### 1.1 目的

1. **跟进优化，BD与销售均可提交跟进，并支持对跟进质量进行管理**
2. **销售项目管理优化，便于获取更准确 销售项目人员归属与收入数据**

### 执行SQL语句

```sql
ALTER TABLE boss_crm.`followup_info` MODIFY `comment` text NOT NULL COMMENT '评分评论';

ALTER TABLE boss_crm.`customer_project` ADD `estimate_release_date` date NULL COMMENT '预计发布时间',
ADD `first_integration_time` datetime NULL COMMENT '集成时间，项目阶段由小于80%变更为大于等于80%的时间',
ADD `first_clue_time` datetime NULL COMMENT '初次线索时间，项目首次变更为线索的时间',
ADD `first_business_time` datetime NULL COMMENT '初次商机时间，项目首次变更为商机的时间',
ADD `assist_sales` varchar(255) NOT NULL DEFAULT '' COMMENT '辅助销售，逗号分割',
ADD `assist_bd` varchar(255) NOT NULL DEFAULT '' COMMENT '辅助BD，逗号分割';

ALTER TABLE boss_crm.`sales_funnel` ADD `delete_time` timestamp NULL DEFAULT NULL COMMENT '删除时间';

CREATE TABLE `customer_project_operate_log` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT '自增id',
  `project_ids` varchar(255) NOT NULL DEFAULT '0' COMMENT '项目id，多个id是批量操作',

  `operate_type` tinyint NOT NULL DEFAULT '0' COMMENT
  '操作类型，1：取消漏斗，2：更新财务，3：更新集团，4：更新bd，5：更新ka，6：批量更新bd，7：批量更新ka',
  `creator` varchar(64) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci NOT NULL DEFAULT '' COMMENT '操作人',
  `created_at` bigint(20) unsigned NOT NULL DEFAULT '0' COMMENT '创建时间',
  `updated_at` bigint(20) unsigned NOT NULL DEFAULT '0' COMMENT '更新时间',
  `extra_content` json NOT NULL DEFAULT '' COMMENT '扩展字段，记录操作详情',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='项目操作记录表';

```

## 会议记录

### 7.1 内部评审会议

改动点

1.有些字段int类型，需要与前端确定枚举值？

A:补充字段定义

2. 线索类型怎么确定，根据什么算法确定？

A:每次更新“转化商机概率”都根据最新提交的 转化商机概率判断一下是 线索还是商机，当判断为 商机 的时候，再根据 成单率 判断这个商机是 商机-常规 还是 商机-赢单

3.日志记录，需要页面展示？

A:从产品维度来看，前端查看是弱需求，加页面会增加整体项目工作量，暂时先以日志方式保存，用作于业务内部排查问题

4.只更新项目bd，ka销售；同步操作只在对应的功能模块修改？

A:保留批量更新，同步操作跳转对应的功能模块修改

### 7.2 设计方案评审

1.新的状态，对旧的记录是否生效?

A: 需要，使用SQL语句更改以前的记录

2.操作记录是否需要保存?

A: 对销售项目的操作需要落库

3.更新操作后，返回数据不一致的展示?

A: 更新接口返回，需要补充数据不一致的财务或集团公司数据
