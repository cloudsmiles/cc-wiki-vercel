## 收支明细

现有接口 /v2/online_sale/get_topup_record 只返回了充值明细的数据，并且是从控制台本地库查的。
新需求需要返回充值明细、公转账、账单扣费、套餐包购买等场景的数据，需统一从manage系统的接口查

### 接口

使用manage系统接口

### 请求参数

| 字段 | 描述 |
| --- | --- |
| page | 当前页数 |
| page_size | 每页条数 |
| date_begin | 开始时间 |
| date_end | 结束时间 |

### 返回参数

| 字段 | 描述 |
| --- | --- |
| time | 当前收支明细发生的时间，按照UTC+ 8的时间显示 |
| 订单号 | 为BOSS系统中的订单ID，没有则显示"-" |
| 交易类型 | 分为在线充值 (PayPal)，在线充值 (Stripe)，Bank Transfer，日结账单预扣费，套餐包充值，套餐包消费，月度扣费（暂时不展示）
-余额扣费，按照每天用户的使用情况进行扣费
-套餐包，将用户购买套餐包的动作分为充值和消费两部分，充值按照订单方式执行（可以参考manage显示） |
| 交易金额 | 有正负数，保留小数点后两位 |

## 收据优化

介绍：

1、目前牵扯到用户收据行为的有套餐包购买、在线充值和对公转账，目前均有对应的邮件通知，需要在邮件通知模板中增加对应的Receipt

2、充值和购买套餐包的行为目前都会在BOOS上记录为订单，这里沿用BOSS订单逻辑进行处理

### 接口

/v2/bill/get_receipt_detail

### 返回参数

| 字段 | 描述 |
| --- | --- |
| Date & Time | 按照实际订单产生的时间显示 |
| 订单号 | 为BOSS系统中的订单ID |
| invoice_title |  |
| 交易类型 | 列表中显示，在购买套餐包时，显示控制台订单的商品名称
-在线充值时显示为Top up
-对用转账时显示为Transfer |
| 支付方式 | 列表中显示，根据实际支付方式返回，分为Stripe、PayPal和Bank transfer |
| 金额 | 列表中显示，根据实际金额返回，保留小数点后两位 |
