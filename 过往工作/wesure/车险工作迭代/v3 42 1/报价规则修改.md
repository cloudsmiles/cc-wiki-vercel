报价前校验规则

RULEA: 判断是否处于“未到报价期”，进行卡片拦截

- 拿最近一笔成交保单得到下次起保时间
    - 保单双险
    - 保单处于待生效 // 拦截
    - 生效中30天内 // 拦截
- 拿最近一笔投保轨迹（报价）得到下次起保时间
    - 投保轨迹双险
    - 创建时间和终保时间相差多于一天
        - 先排除创建时间为2月28日和3月1日的情况，此种情况要考虑2月29日
        - 到期时间同步到创建时间的同一年份
        - 再计算天数的差值
    - 创建日期距离当前日期少于750天
- 根据下次起保时间计算出窗口期开始时间
- 判断区间
    - 区间 [窗口期开始时间, 窗口期结束时间+30days]
    - 不处在区间，说明处于“未到报价期”
- 处于“未到报价期”，进行拦截

- [x]  投保轨迹-通过生效时间计算到期时间
- [x]  轨迹ct和保单ct大小比较，通过到期时间而非下笔保单生效时间
- [x]  取最新一笔逻辑变更
- [x]  相差一天计算重构
